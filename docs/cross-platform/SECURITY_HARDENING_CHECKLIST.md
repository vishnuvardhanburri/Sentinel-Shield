# Cross-Platform Security Hardening Checklist

## Shared Rules

- No secrets in web, desktop, or mobile clients.
- No direct client access to Ollama, Redis, Postgres, Chroma, or ledger files.
- All policy, DLP, prompt-injection, routing, risk, and quarantine decisions are server-side.
- Tokens are short-lived access tokens plus rotated refresh tokens.
- Every client request includes device context.
- All high-risk actions are written to the Obsidian ledger.

## Web

- Hardened CSP in the Next.js shell.
- Strict CORS enforced by backend `ALLOWED_ORIGINS`.
- No wildcard origin allowed in production.
- Browser storage is limited to tokens and non-sensitive UI state.
- Evidence export is downloaded from the backend only after RBAC authorization.

## Desktop: Tauri

- Secure IPC boundary with explicit command registration only.
- No arbitrary shell execution exposed to frontend JavaScript.
- Native notifications are enabled for CISO alerts.
- Updater channel is configured but requires buyer-controlled signing keys and update endpoint.
- macOS notarization, Windows signing, and Linux package signing are release-time requirements.
- Sensitive exports are generated by the backend, not assembled locally.

## Mobile: React Native

- Encrypted token storage via platform secure storage.
- Certificate pinning required for production native builds.
- Push alerts must contain no PII and should fetch incident details after authentication.
- Emergency kill switch actions require server-side RBAC and ledger audit.
- Read-only audit trail by default; write actions require elevated roles.

## Backend

- Fail-closed environment secret validation.
- JWT + refresh-token rotation.
- Device session tracking and revocation.
- Rate limiting and request size limits.
- Secure headers and suspicious path blocking.
- mTLS support through buyer-controlled reverse proxy.
- Tamper-evident JSONL audit ledger.

## Release Gate

Before shipping a cross-platform release:

```bash
pnpm submit:ready
pnpm cross:check
pnpm generate:data-room
```

Then validate platform builds through `.github/workflows/cross-platform-release.yml`.
